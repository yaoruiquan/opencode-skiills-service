## 三、Job 文件模型

```
/data/work/jobs/{job_id}/
├── input/          ← 调用方上传的材料
├── output/         → skill 生成的可下载成果
├── logs/           → OpenCode stdout/stderr/attempt 日志
└── job.json        　任务状态、模板、文件列表、运行参数
```

约束：prompt 中只引用容器路径，不允许传 macOS 绝对路径。

## 四、任务生命周期

```
  ┌──────────┐    ┌──────────┐    ┌──────────┐    ┌──────────┐    ┌──────────┐
  │  CREATED  │───→│ RUNNING  │───→│SUCCEEDED │    │ RETRYING │───→│  FAILED  │
  └──────────┘    └──────────┘    └──────────┘    └──────────┘    └──────────┘
                       │                                  ↑
                       │    capacity error                 │  still capacity
                       └────────────→  重试/切 fallback  ──┘  error → FAILED
```

## 五、API 流程

```
POST /jobs                    GET /jobs/{id}
  ├─ resolve template           ├─ status / files / run info
  ├─ create job_{uuid}          └─ 轮询直到 succeeded/failed
  ├─ mkdir input/output/logs
  └─ write job.json           GET /jobs/{id}/logs
                                  ├─ run.jsonl (stdout)
POST /jobs/{id}/files            └─ stderr.log
  ├─ validate relative path
  ├─ reject `..`/NUL/abs       GET /jobs/{id}/outputs[/{path}]
  └─ write to input/             └─ 列出 / 下载 output 文件

POST /jobs/{id}/run
  ├─ validate required inputs
  ├─ generate template prompt        GET /health
  ├─ status → running                  ├─ template list
  ├─ async: opencode run              ├─ default model
  └─ return 202 immediately           └─ fallback config
```

## 六、模板系统

```
         TEMPLATES {}
           ├─ label / description / inputMode
           ├─ requiredInputs / recommendedInputs
           ├─ modes [single, batch, ...]
           ├─ browserMcp / chromePort (上报类)
           └─ outputs []

生成 prompt ──→ md2wechatPrompt(job)       固定路径 + 脚本命令
             └─ complexSkillPrompt(job, template, body, inputFiles)
                    ├─ shared(路径/参数/规则)
                    └─ specifics[template]
```

## 七、OpenCode 重试逻辑

```
modelCandidates = [...requested, DEFAULT_MODEL, ...FALLBACK_MODELS]
                        │
buildAttemptPlan(models):
  ┌─ 首选模型: CAPACITY_RETRIES + 1 次
  └─ 后续模型: 各 1 次

attempt loop:
  ├─ 检测 stdout/stderr 是否包含 capacity 错误
  ├─ 是 → 标记 retrying，继续下一次 attempt
  └─ 否 → 标记 failed，立即停止
```

## 九、安全边界

```
┌────────────────────────────────────────────┐
│  ✅ 已实现                                  │
│  .env 不提交                               │
│  job 路径固定，相对路径校验 + 拒绝 path traversal │
│  output 下载限制在 job 目录内                  │
│  Chrome DevTools 绑定 127.0.0.1             │
├────────────────────────────────────────────┤
│  ❌ 待补（当前仅限本地可信环境）                 │
│  无鉴权 / 无队列 / 无并发限制 / 无租户隔离        │
│  无 SSE/WebSocket 日志流 / 无监控告警          │
└────────────────────────────────────────────┘
```

## 十、目录职责

```
opencode-skills-service/
├── backend/            服务化边界（API / 模板 / prompt / job 管理）
├── frontend/           纯静态中文控制台，无构建步骤
├── opencode-server/    OpenCode 运行镜像 + 启动脚本
├── browser-service/    Docker Chrome 镜像 + nginx 代理
├── config/             OpenCode 配置模板（.env 渲染）
├── scripts/            本地辅助脚本
├── progress/           项目规划/进展/实现说明
└── data/               运行数据 + Chrome profile（仅 .gitkeep 入库）
```
