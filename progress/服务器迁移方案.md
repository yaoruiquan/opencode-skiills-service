# 服务器迁移方案

本文档给出服务器试迁移方案。目标是把本地已验证的 OpenCode + skills-api + frontend + Docker Chrome 链路迁移到 Linux 服务器。

## 1. 推荐目录

```text
/opt/opencode-skills-service              # 本服务项目
/opt/opencode-skills/skills               # .claude/skills Git 仓库
/var/lib/opencode-skills/work             # job input/output/logs
/var/lib/opencode-skills/output           # 预留最终输出
/var/lib/opencode-skills/docker-chrome-profiles
/etc/opencode-skills/.env                 # 服务器环境变量，权限 600
```

当前 compose 仍默认使用项目内 `./data/*`。服务器试迁移有两种方式：

1. 先用项目内 `./data/*` 跑通，简单但不利于长期维护。
2. 增加服务器 override，把 `data/work`、`data/output`、Chrome profile 映射到 `/var/lib/opencode-skills/*`。

建议试迁移可以先用方案 1；稳定后切到方案 2。

## 2. 服务器准备

推荐：

- Ubuntu 22.04 或 24.04。
- Docker 和 Docker Compose plugin。
- 4 CPU cores minimum。
- 8 GB RAM minimum，16 GB recommended。
- 100 GB disk recommended。
- 能访问 DeepSeek API、Git 仓库、Debian/Node/Python 包镜像。

基础命令：

```bash
sudo mkdir -p /opt/opencode-skills-service
sudo mkdir -p /opt/opencode-skills/skills
sudo mkdir -p /var/lib/opencode-skills/work
sudo mkdir -p /var/lib/opencode-skills/output
sudo mkdir -p /var/lib/opencode-skills/docker-chrome-profiles
sudo mkdir -p /etc/opencode-skills
```

## 3. 同步服务项目

当前本地 `opencode-skills-service` 没有 Git remote。服务器要 `git pull` 之前，需要先把本项目推到 Git 仓库。

如果已有远端仓库：

```bash
cd /opt
git clone <opencode-skills-service-git-url> opencode-skills-service
cd /opt/opencode-skills-service
```

如果暂时没有远端仓库，可以先用 `rsync` 做首次试迁移，但长期仍建议补 Git remote。

## 4. 同步 skills 仓库

业务 skills 源仓库是：

```text
/Users/yao/.claude/skills
```

不是：

```text
/Users/yao/.agents
```

`.agents` 是 agent/opencode 等工具管理目录，不作为业务 Git 源。

本地 skills 仓库当前可选 remote：

```text
origin    https://github.com/yaoruiquan/vulns_skills.git
ai-skills http://gitlab.info.dbappsecurity.com.cn/steven.king/AI-Skills.git
gitlab    http://gitlab.info.dbappsecurity.com.cn/ruiquan.yao/vulnerability-operations
```

服务器上选择一个正式 remote，例如：

```bash
cd /opt/opencode-skills
git clone <skills-git-url> skills
cd /opt/opencode-skills/skills
git checkout main
```

后续更新：

```bash
cd /opt/opencode-skills/skills
git pull
```

## 5. 服务器 `.env`

服务器 `.env` 建议放在：

```text
/etc/opencode-skills/.env
```

然后复制到项目根目录或用 symlink：

```bash
cd /opt/opencode-skills-service
ln -s /etc/opencode-skills/.env .env
```

示例：

```env
OPENCODE_IMAGE=ghcr.io/anomalyco/opencode:latest

OPENCODE_PROVIDER_ID=deepseek-anthropic
OPENCODE_PROVIDER_NAME=DeepSeek Anthropic
OPENCODE_PROVIDER_NPM=@ai-sdk/anthropic
OPENCODE_API_BASE_URL=https://api.deepseek.com/anthropic
OPENCODE_API_KEY=replace-with-your-deepseek-api-key

OPENCODE_MODEL=deepseek-v4-flash
OPENCODE_MODEL_DISPLAY_NAME=DeepSeek V4 Flash
OPENCODE_MODEL_REASONING=false
OPENCODE_MODEL_CONTEXT=128000
OPENCODE_MODEL_OUTPUT=64000

SKILLS_HOST_DIR=/opt/opencode-skills/skills

CHROME_DEVTOOLS_CNVD_HOST=browser-cnvd
CHROME_DEVTOOLS_CNNVD_HOST=browser-cnnvd
CHROME_DEVTOOLS_NCC_HOST=browser-ncc
CHROME_DEVTOOLS_CNVD_PORT=9332
CHROME_DEVTOOLS_CNNVD_PORT=9333
CHROME_DEVTOOLS_NCC_PORT=9334

SKILLS_API_PORT=4100
SKILLS_API_JOBS_ROOT=/data/work/jobs
OPENCODE_FALLBACK_MODELS=deepseek-anthropic/deepseek-v4-pro
OPENCODE_CAPACITY_RETRIES=1
OPENCODE_CAPACITY_RETRY_DELAY_MS=5000
```

权限：

```bash
chmod 600 /etc/opencode-skills/.env
```

## 6. 可选服务器 override

如果要把运行数据放到 `/var/lib/opencode-skills`，新增：

```text
docker-compose.server.yml
```

内容：

```yaml
services:
  opencode-server:
    volumes:
      - ${SKILLS_HOST_DIR:-/opt/opencode-skills/skills}:/root/.agents/skills:ro
      - ./config/opencode.template.json:/opt/opencode/templates/opencode.template.json:ro
      - opencode-config:/root/.config/opencode
      - /var/lib/opencode-skills/work:/data/work
      - /var/lib/opencode-skills/output:/data/output
      - opencode-state:/root/.local/share/opencode

  skills-api:
    volumes:
      - ./backend:/app:ro
      - /var/lib/opencode-skills/work:/data/work
      - /var/lib/opencode-skills/output:/data/output

  browser-cnvd:
    volumes:
      - /var/lib/opencode-skills/docker-chrome-profiles/cnvd-report:/home/chrome/profile

  browser-cnnvd:
    volumes:
      - /var/lib/opencode-skills/docker-chrome-profiles/cnnvd-report:/home/chrome/profile

  browser-ncc:
    volumes:
      - /var/lib/opencode-skills/docker-chrome-profiles/ncc-report:/home/chrome/profile
```

启动：

```bash
docker compose -f docker-compose.yml -f docker-compose.server.yml --profile docker-browser up -d --build opencode-server skills-api frontend browser-cnvd browser-cnnvd browser-ncc
```

## 7. CNVD 上报迁移要点

参考 `phase2-cnvd-report` skill，本地原始说明依赖：

- macOS Chrome profile。
- 本机 Chrome debug port `9332`。
- skill 自己的 `.env`。
- 可选上传服务器 `10.50.10.29`。
- 可选钉钉 webhook。

迁移到本项目后，浏览器层改为：

```text
OpenCode -> chrome-devtools-mcp -> browser-cnvd:9332 -> Docker Chrome internal 9222
```

服务器上不再运行 `scripts/start-chrome-debug.sh` 启动 macOS Chrome。该脚本只作为旧本地模式或 fallback 参考。

仍需保留或迁移的配置：

- `phase2-cnvd-report/.env` 中的平台账号或 profile 策略。
- 上传服务器 SSH key。
- `REPORT_UPLOAD_HOST` / `REPORT_UPLOAD_USER` / `REPORT_UPLOAD_REMOTE_DIR`。
- 钉钉 webhook 和 secret。
- OCR 依赖。

不要把以下内容写进本服务仓库：

- CNVD 账号密码。
- 上传服务器密码。
- SSH 私钥。
- 钉钉 webhook。
- 平台 cookies。
- Chrome profile。

## 8. 启动和验收

启动：

```bash
cd /opt/opencode-skills-service
docker compose --profile docker-browser up -d --build opencode-server skills-api frontend browser-cnvd browser-cnnvd browser-ncc
```

基础验收：

```bash
curl -sS http://127.0.0.1:4096/global/health
curl -sS http://127.0.0.1:4096/mcp
curl -sS http://127.0.0.1:4100/health
curl -sS http://127.0.0.1:19332/json/version
curl -sS http://127.0.0.1:19333/json/version
curl -sS http://127.0.0.1:19334/json/version
```

skills 挂载验收：

```bash
docker compose exec -T opencode-server sh -lc 'test -f /root/.agents/skills/md2wechat/SKILL.md'
docker compose exec -T opencode-server sh -lc 'test -f /root/.agents/skills/phase2-cnvd-report/SKILL.md'
docker compose exec -T opencode-server sh -lc 'test -f /root/.agents/skills/msrc-vulnerability-report/SKILL.md'
docker compose exec -T opencode-server sh -lc 'test -f /root/.agents/skills/cnvd-weekly-db-update/SKILL.md'
```

第一轮业务验收：

1. `custom` 写文件 smoke。
2. `md2wechat` 真实输出。
3. `phase2-cnvd-report` 只跑 `prepare_form_context.py`，不提交。
4. `cnvd-weekly-db-update` 只跑 `check`，不跑 `update`。

## 9. 回滚

试迁移阶段回滚方式：

```bash
docker compose --profile docker-browser down
```

如果使用 `/var/lib/opencode-skills`，不要直接删除数据目录。先备份：

```bash
tar -czf /tmp/opencode-skills-backup-$(date +%Y%m%d-%H%M%S).tgz /var/lib/opencode-skills
```
