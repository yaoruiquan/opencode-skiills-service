# 本地 Agent + Skills 到服务器服务化的差异与收敛计划

## 背景

本地 Claude Code + skills 能跑得顺，是因为它运行在熟悉的本机环境里：浏览器登录态、Cookie、文件路径、人工观察和临场修正都在同一个交互式会话中完成。服务器迁移后，执行链路变为：

```text
Frontend -> skills-api -> OpenCode run -> skill -> Docker Chrome -> 平台
```

这条链路要求每个步骤都可被服务端观测、恢复和校验。仅同步 skill 文件不能保证等价运行。

## 关键差异

| 维度 | 本地 Claude Code + skills | 服务器 OpenCode + skills-api |
| --- | --- | --- |
| 执行方式 | 交互式 agent，可即时纠错 | 后台任务，依赖固定输入、输出、日志和状态 |
| 浏览器环境 | 本机 Chrome，复用日常登录态 | Docker Chrome，需要独立 profile 初始化和保活 |
| 人工兜底 | 人能直接看浏览器、输验证码、修路径 | 必须通过前端截图、人工输入文件和状态事件传递 |
| 模型行为 | Claude Code 长流程遵循和工具修正更强 | OpenCode + DeepSeek 更需要硬契约约束 |
| 成功判断 | 人看平台页面即可确认 | 后端必须从输出文件、平台编号、状态机判断 |
| 错误恢复 | agent 可临场调整 | 服务端必须有明确失败格式和可重跑边界 |

## 收敛原则

1. 只保留 `OpenCode + skills` 主路径。
   - deterministic adapter 只作为 legacy/可选路径。
   - 默认不启用、不继续扩展 adapter。

2. 每个 skill 必须有服务化执行契约。
   - 固定输入目录：`input/`
   - 固定输出目录：`output/`
   - 固定日志目录：`logs/`
   - 固定配置文件：`input/service-config.json`
   - 固定进度文件：`logs/progress.jsonl`
   - 固定失败表达：`output/summary.txt` 或 `output/submission-result.json`

3. 上报类 skill 必须使用任务级状态机。
   - `prepare`
   - `form_context`
   - `browser`
   - `login`
   - `cloudflare`
   - `captcha`
   - `fill_form`
   - `upload`
   - `submit`
   - `extract_id`
   - `summary`

4. Docker Chrome profile 必须独立管理。
   - CNVD、CNNVD、NCC 使用独立 profile。
   - profile 初始化、登录态刷新和健康检查单独记录。
   - 不把本地 macOS Chrome profile 当作服务器运行前提。

5. 成功结果优先于 job 状态。
   - 如果 `output/submission-result.json` 中已有平台编号和成功状态，应视为平台提交成功。
   - 即使 job 后续被 cancel、failed 或重启覆盖，前端也必须显示“平台已提交”。
   - 平台编号、submission URL、提交时间是最高优先级证据。

## 当前执行要求

后续开发和排障统一遵循以下判断顺序：

1. 先看 `executionMode`，必须是 `opencode-skills`。
2. 再看 `output/submission-result.json`，若存在成功平台编号，优先认定平台成功。
3. 再看 `logs/progress.jsonl`，定位卡在哪个业务阶段。
4. 再看 `logs/run.jsonl` 和 `logs/stderr.log`，确认 OpenCode 是否漏写输出或改写配置。
5. 最后才看 job 原始 `status`。

## 下一步实施

- 后端：继续默认禁用 adapter，并在 health 中暴露执行模式。
- 后端：把 skill 服务化契约写入 prompt，禁止 OpenCode 重写 `service-config.json`。
- 后端：对 submission 成功结果做优先判定。
- 前端：任务列表和当前任务状态显示 `effectiveStatus`，不要只看原始 job status。
- 浏览器：维护 Docker Chrome profile 初始化和健康检查文档。
