## 5. 迁移前风险

### 5.1 服务仓库没有 remote

当前 `opencode-skills-service` 本地仓库没有配置 Git remote。服务器如果要 `git pull` 这个项目，需要先创建远端仓库并推送，或者用 `rsync`/压缩包方式做首次同步。

### 5.2 skills 仓库有多个 remote

`/Users/yao/.claude/skills` 是 Git 仓库，当前分支：

```text
main
```

当前提交：

```text
ccf3b13
```

已配置 remote：

```text
origin  https://github.com/yaoruiquan/vulns_skills.git
ai-skills  http://gitlab.info.dbappsecurity.com.cn/steven.king/AI-Skills.git
gitlab  http://gitlab.info.dbappsecurity.com.cn/ruiquan.yao/vulnerability-operations
```

服务器迁移前需要明确使用哪个 remote 作为正式同步源。

### 5.3 CNVD 上报配置需要服务器化

CNVD skill 原本的本地说明依赖 macOS Chrome 和本地 profile。迁移后应改为使用本项目的 Docker Chrome：

```text
browser-cnvd:9332
```

账号、上传服务器、钉钉和 SSH key 仍应配置在服务器的 skill `.env` 或密钥系统中，不要写入本服务仓库。
