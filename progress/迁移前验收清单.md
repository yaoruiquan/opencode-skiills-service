# 迁移前验收清单

本文档用于判断本地环境是否已经达到服务器试迁移条件。

## 1. 本地服务状态

必过项：

```bash
docker compose config --quiet
curl -sS http://127.0.0.1:4096/global/health
curl -sS http://127.0.0.1:4096/mcp
curl -sS http://127.0.0.1:4100/health
curl -sS http://127.0.0.1:19332/json/version
curl -sS http://127.0.0.1:19333/json/version
curl -sS http://127.0.0.1:19334/json/version
```

当前判断：

- OpenCode Server 必须健康。
- `skills-api` 必须健康。
- 三路 MCP 必须 `connected`。
- 三路 Docker Chrome 必须能返回 `/json/version`。

## 2. `md2wechat` 真实 workflow

验收目标：

- 通过 API 或前端创建 `md2wechat` job。
- 写入 `input/article.md`。
- 运行模板。
- 生成以下输出：

```text
output/wechat-article.html
output/wechat-cover.png
logs/render_wechat_article.json
```

用户已确认：本地 `md2wechat` 可以输出。

迁移前还需要记录一次 job id、输出文件列表和是否需要调整样式或封面规则。

## 3. 复杂 skill smoke test

迁移服务器前至少选择一个复杂 skill 做最小 smoke test。

推荐顺序：

1. `phase1-material-processor`：不依赖浏览器平台登录，适合验证材料目录和 docx 处理。
2. `phase2-cnvd-report`：验证 CNVD MCP 通道和浏览器打开目标页。
3. `phase2-cnnvd-report` 或 `phase2-ncc-report`：验证其它浏览器 profile。

验收不要求真实提交平台表单，但至少要确认：

- 输入材料能通过 job input 传入。
- prompt 中没有 macOS 绝对路径。
- skill 能读取 `/root/.agents/skills/{skill_name}`。
- 如果涉及浏览器，MCP 通道能打开页面或完成准备阶段。
- 输出或 summary 写入 job output/logs。

## 4. Skills 同步策略

本项目不 vendor skills。服务器迁移前必须确认 skills 同步方式。

### 4.1 当前服务化迁移范围

前端展示的模板来自 `skills-api /health`，不是扫描完整 skills 仓库。当前已按 `/Users/yao/.claude/skills/README.md` 中的业务清单服务化 8 个 skill：

| 前端模板 | skill 目录 | 状态 |
| --- | --- | --- |
| 公众号转换 | `md2wechat` | 已接入，用户已确认本地可输出 |
| 漏洞预警材料 | `vulnerability-alert-processor` | 已接入模板，待真实 smoke test |
| 材料整理 | `phase1-material-processor` | 已接入模板，推荐作为复杂 skill 首个 smoke test |
| MSRC 预警报告 | `msrc-vulnerability-report` | 已接入模板，待材料包 smoke test |
| CNVD 周库更新 | `cnvd-weekly-db-update` | 已接入模板，真实 update 前必须确认 SSH/Docker/钉钉配置 |
| CNVD 上报 | `phase2-cnvd-report` | 已接入模板，依赖 CNVD Docker Chrome |
| CNNVD 上报 | `phase2-cnnvd-report` | 已接入模板，依赖 CNNVD Docker Chrome |
| NCC 上报 | `phase2-ncc-report` | 已接入模板，依赖 NCC Docker Chrome |

此外后端还有 `custom` 模板，但它不是具体 skill。

`.claude/skills` 仓库中还有很多工程类、规范类、模式类 skill，它们可以随 Git 仓库同步到服务器，但当前不作为前端业务任务模板展示。后续如果要服务化新的业务 skill，需要在 `backend/server.js` 注册模板并补充 prompt/测试。

当前本地：

```env
SKILLS_HOST_DIR=/Users/yao/.claude/skills
```

推荐服务器：

```env
SKILLS_HOST_DIR=/opt/opencode-skills/skills
```

可选同步方案：

1. 服务项目正式源：`https://github.com/yaoruiquan/opencode-skiills-service.git`。
2. skills 正式源：`https://github.com/yaoruiquan/vulns_skills.git`。
3. 服务器上两个仓库都使用 `git pull origin main` 更新。
4. 不使用 `/Users/yao/.agents` 作为业务 skills 来源。

同步后必验：

```bash
docker compose exec -T opencode-server sh -lc 'test -f /root/.agents/skills/md2wechat/SKILL.md'
docker compose exec -T opencode-server sh -lc 'test -f /root/.agents/skills/phase2-cnvd-report/SKILL.md'
curl -sS http://127.0.0.1:4100/health
```

如果 skill 更新引入新系统依赖，需要同步更新运行镜像，而不是只同步 skill 文件。

## 5. 数据和密钥边界

迁移前确认不进入 Git：

- `.env`
- DeepSeek API key
- Chrome profile
- job input/output/logs
- 平台 cookies、session、验证码中间数据
- skill 运行缓存

服务器上建议预留：

```text
/opt/opencode-skills-service
/opt/opencode-skills/skills
/var/lib/opencode-skills/work
/var/lib/opencode-skills/output
/var/lib/opencode-skills/docker-chrome-profiles
```

## 6. 迁移结论

满足以下条件后，可以进入服务器试迁移：

- [ ] 本地基础服务检查全部通过。
- [ ] `md2wechat` 真实 workflow 输出已确认。
- [ ] 至少一个复杂 skill smoke test 通过。
- [ ] skills 同步方式已确定。
- [ ] 服务器目录、`.env`、profile 和 job 数据策略已确定。
- [ ] README 和实现逻辑文档已同步当前路径和启动命令。

未满足时，不建议直接迁移生产环境。
